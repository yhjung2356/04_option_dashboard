## 🧮 구현된 계산 로직

### 1. Put/Call Ratio
```
P/C Ratio = Put 거래량 / Call 거래량
- < 0.7: 강세 시장 (콜 우세)
- 0.7~1.0: 중립
- > 1.0: 약세 시장 (풋 우세)
```

### 2. Max Pain
```
Max Pain = 모든 행사가별로
  Σ(콜 intrinsic value × 콜 OI) + 
  Σ(풋 intrinsic value × 풋 OI)
  가 최소가 되는 가격
```

### 3. Delta 계산
```
Call Delta: 0 ~ 1 (OTM → ITM)
Put Delta: -1 ~ 0 (ITM → OTM)
ATM Delta ≈ ±0.5
```

### 4. 옵션 가격
```
Option Price = Intrinsic Value + Time Value
Intrinsic Value:
  - Call: max(0, 현재가 - 행사가)
  - Put: max(0, 행사가 - 현재가)
```

## 💡 구현 아이디어 상세

### 1. 일반 HTS 기능 (기본)
- 선물/옵션 전체 거래량
- 거래대금 상위 종목
- 실시간 가격

### 2. 추가한 디테일한 기능들

#### A. Put/Call Ratio (시장 심리)
투자자들이 풋옵션을 더 많이 사는지, 콜옵션을 더 많이 사는지 비율로 표시
→ 시장의 방향성 예측 가능

#### B. Max Pain
옵션 매도자(기관투자자)가 최소 손실을 보는 가격
→ 만기일에 이 가격으로 수렴하는 경향

#### C. 행사가별 거래 집중도
어떤 행사가에 거래가 몰리는지 실시간 표시
→ 시장 참여자들의 심리 파악

#### D. ATM 강조
현재가 근처 행사가를 강조 표시
→ 가장 활발한 거래가 일어나는 구간

#### E. Greeks 표시
델타, 감마, 세타, 베가 등 옵션 민감도 지표
→ 리스크 관리 및 헷지 전략 수립

#### F. 호가 스프레드
매수/매도 호가 차이 표시
→ 유동성 및 거래 비용 파악
